# Завершить сессию (end-session)

Когда пользователь просит завершить текущую сессию (или вызывает этот навык), выполни следующие шаги:

## 1. Обновление лога сессии
Создай или обнови лог в `logs/sessions/YYYY-MM-DD.md` (используй текущую дату) со следующей структурой:
- Что запрашивалось в ходе сессии
- Что было прочитано и изучено
- Что было сделано (реализовано)
- Какие файлы были изменены
- Принятые решения (особенно архитектурные)
- Открытые вопросы и блокеры
- **Lessons Learned (Выводы для будущих агентов):** Опиши 1-2 урока, ошибки или инсайта, которые ты получил в этой сессии, чтобы следующий агент их не повторял. (Если урок критически важный, запиши его в `docs/wiki/AGENTS_KNOWLEDGE.md`)
- Следующий шаг для будущей сессии

## 2. Обновление Changelog
Обнови `docs/outputs/CHANGELOG.md` (если в течение сессии был изменен исходный код).

## 3. Обновление Decisions
Обнови `docs/wiki/DECISIONS.md` (если в ходе сессии принимались новые архитектурные решения).

## Ожидаемый ответ
После успешного обновления всех необходимых файлов предоставь краткий финальный отчет по следующему шаблону:
- Что было сделано за сессию (2-3 предложения)
- Какие файлы были обновлены (список)
- Команды для финальной проверки (например, `cd src && flutter analyze`)
- Следующий шаг для продолжения работы
