# Project Context — DnevnikFlutter

## 🎯 Что это

Мобильное Flutter-приложение «Школьный Дневник» — личный помощник-чит для школьников.
Позволяет видеть расписание уроков, управлять домашними заданиями и получать помощь с учёбой через встроенный AI-ассистент, цель сделать приложение незаменимым для ученика путем стильного дизайна и необыкновенно удобного интерфейса для помощи и поиску ответов на школьные задания.
Приложение работает в системе классов: ученик создаёт или вступает в класс по коду, после чего получает доступ к общему расписанию и ДЗ своего класса.

## 🔧 Почему существует

Стандартные школьные журналы неудобны иногда не работают у всех долго обновляются, не дают быстрого доступа к нужной информации.
Это приложение решает проблему: ученик за секунды видит, что задано и на когда, может за секунду сфотографировать задание и добавить его через AI-парсинг текста, а также задать вопрос ассистенту прямо в контексте своего расписания и заданий.

- **Хранение секретов:** flutter_secure_storage ^10.3.1, flutter_dotenv ^6.0.1
- **Локальный кэш:** shared_preferences ^2.2.2
- **Прокси:** Vercel (node_modules/vercel-proxy в `src/vercel-proxy/`)
- **Deploy:** Android APK вручную (сборка локально, распространение напрямую)

## 🎨 Основные фичи (реализованные)

1. **Система классов** — создание класса (admin) или вступление по коду; расписание и ДЗ изолированы по классу
2. **Расписание** — недельное расписание уроков с временем, кабинетом и цветовой маркировкой предметов
3. **Домашние задания** — просмотр, добавление (вручную + через AI-парсинг текста), отметка выполненных, фото-вложения
4. **AI-ассистент** — чат с Gemini в контексте расписания и ДЗ, поддержка изображений (фото задания → объяснение)
5. **Умное добавление ДЗ** — фраза типа «Алгебра на завтра номера 45 46» → автоматически парсится в структурированное задание
6. **Оффлайн-кэш** — ДЗ кэшируются в памяти и на диске (SharedPreferences), работает без сети
7. **Аутентификация** — Firebase Auth, токены хранятся в flutter_secure_storage
8. **Панель администратора** — управление расписанием и настройками класса

## 🔮 Планируемые фичи

- [ ] **Моментальные ответы на задачи** — режим «реши задачу», где ученик фотографирует или вводит номер/условие и AI сразу даёт пошаговое решение
- [ ] **Редизайн UI** — стильный, современный интерфейс; акцент на удобство одной рукой, быстрый доступ к нужным данным
- [ ] **Интеграция с гос. электронным дневником** — синхронизация оценок из официального журнала (ЭПОС / Дневник.ру / регионального аналога)
- [ ] **AI-анализ успеваемости** — ИИ видит оценки ученика и даёт персональные советы: «До 4 по Алгебре не хватает X — на понедельник лёгкое задание, давай подготовимся вместе»
- [ ] **Персональный план улучшения оценок** — ИИ составляет конкретный план действий с учётом расписания и текущих пробелов

## 🗄️ Структура данных Firestore

```
/classes/{classId}/
    className: string
    schoolName: string
    code: string           # Код для вступления
    schedule: map          # Расписание {0: [...], 1: [...]} (0=Пн, 4=Пт)
    subjects: array        # Список предметов класса
    lessonTimes: array     # Время уроков

/classes/{classId}/homework/{hwId}/
    id: string
    subject: string
    task: string
    deadline: string       # "YYYY-MM-DD"
    imageUrl: string?      # (устаревшее, мигрируется в imageUrls)
    imageUrls: array?      # Base64 превью изображений
    fullResolutionUrls: array?  # Base64 полноразмерных изображений
    fromSchedule: bool
```

## 🚫 Ограничения

- Не менять `.env` — содержит Firebase API ключи
- Не менять `firestore.rules` без явного разрешения пользователя
- Не добавлять зависимости в `pubspec.yaml` без одобрения
- Firestore используется через REST API напрямую (не FlutterFire) — учитывать при работе с данными
- AI-запросы идут через Vercel-прокси, не напрямую — при изменении API учитывать прокси
