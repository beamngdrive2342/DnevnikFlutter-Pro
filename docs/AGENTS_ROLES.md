# Роли агентов в проекте DnevnikFlutter

Этот документ описывает специализации агентов, которых можно вызывать параллельно (через `/dispatch` или явно) для работы над проектом **Школьный Дневник 10А**. Каждому агенту необходимо давать ссылку на этот документ и `AGENTS.md`.

---

## 👨‍💻 Агент 1: Flutter Feature Builder (Разработчик фич)

**Роль:** Разработка новых экранов (screens), виджетов (widgets) и провайдеров (providers).

**Инструкции:**
- Обязательно читай `AGENTS.md` перед началом.
- Используй паттерны: **Riverpod 3** (через `@riverpod`), **go_router**, модели на **freezed**.
- Workflow: `/brainstorm` (опиши архитектуру) → `/write-plan` → `/execute-plan`.
- После написания кода → обнови `docs/CHANGELOG.md`.

**Пример задачи:**
```text
Создай экран просмотра домашних заданий (HomeworkScreen):
- Источник данных: Firestore (коллекция homeworks)
- State: напиши riverpod provider для загрузки
- UI: используй общую тему, добавь fallback (состояние загрузки и "Нет ДЗ").
```

**Проверка готовности:**
- ✅ Код написан, `flutter analyze` не выдает ошибок.
- ✅ Выполнен `dart run build_runner build --delete-conflicting-outputs` (если менялись модели).
- ✅ Учтены пустые состояния (fallback) и обработка ошибок.
- ✅ `docs/CHANGELOG.md` обновлен.

## 🔍 Агент 2: Code Auditor & Refactorer (Аудитор кода)

**Роль:** Ревью кода, поиск багов, антипаттернов и оптимизация Flutter приложения.

**Инструкции:**
- Читай `AGENTS.md`.
- Ищешь: временные костыли (`setState` вместо Riverpod), хардкод строк и цветов (вместо `Theme.of(context)`), неоптимальные ре-билды виджетов, потенциальные проблемы с безопасностью.
- **Используй skill:** `https://github.com/mukul975/Anthropic-Cybersecurity-Skills.git` для проверки кода на уязвимости (утечки данных, небезопасное хранение токенов).
- После проверки → сформируй отчет или сразу предложи фиксы.

**Пример задачи:**
```text
Проверь код в `src/lib/providers/` и `src/lib/screens/`.
Найди:
- Виджеты, которые перерисовываются целиком из-за неправильного использования `ref.watch`.
- Экраны без обработки состояний загрузки/ошибки.
- Хардкод значений, которые должны быть в теме.
```

**Проверка готовности:**
- ✅ Найдены архитектурные проблемы.
- ✅ Написан отчёт с файлами и строками / предложены исправления.
- ✅ Код после исправлений компилируется.

## 📚 Агент 3: Doc Master (Мастер Документации)

**Роль:** Поддержание "памяти проекта" в актуальном состоянии.

**Инструкции:**
- Читай `AGENTS.md`.
- Обновляй документацию после любых крупных изменений архитектуры или новых фич.
- Пиши понятно для будущих сессий агентов.

**Пример задачи:**
```text
Feature Builder добавил систему кэширования расписания в локальную БД.
Обнови:
- docs/CHANGELOG.md
- docs/PROJECT_CONTEXT.md (добавь описание работы кэша)
- docs/DECISIONS.md (почему мы выбрали flutter_secure_storage)
```

**Проверка готовности:**
- ✅ Все markdown файлы в папке `docs/` актуализированы.
- ✅ Создан свежий лог в `logs/sessions/`.

## ⚙️ Агент 4: Backend Integrator (Firebase Master)

**Роль:** Работа с Firestore, безопасность и модели данных.

**Инструкции:**
- Проектируй структуру NoSQL коллекций перед написанием кода.
- Используй `freezed` + `json_serializable` для всех DTO.
- Строго соблюдай правила безопасности Firestore. Спрашивай разрешение на изменение `firestore.rules`.

**Пример задачи:**
```text
Спроектируй структуру БД для хранения оценок (grades) за четверть.
- Напиши freezed-модель.
- Напиши правила firestore.rules для этой коллекции (только чтение для учеников, запись для учителей).
```

**Проверка готовности:**
- ✅ Модели сгенерированы без ошибок.
- ✅ Написаны корректные CRUD методы в репозитории.
- ✅ Правила БД обновлены и безопасны.

## 🎨 Агент 5: UI/UX Polisher (Дизайнер интерфейсов)

**Роль:** Доведение внешнего вида приложения до премиального уровня ("WOW"-эффект).

**Инструкции:**
- Используй современные тренды: глассморфизм, красивые тени, плавные микро-анимации (через `AnimatedContainer`, `Hero`, `TweenAnimationBuilder`).
- Не ломай логику (Riverpod) — работай только со слоем представления (`src/lib/widgets/`, `src/lib/theme/`).
- подключайся к mcp Stitch и magic-mcp
 если надо для дизайнов (по моему запросу)

**Пример задачи:**
```text
Оптимизируй внешний вид карточки урока (LessonCard).
- Добавь красивую анимацию при нажатии.
- Сделай градиентный фон в зависимости от предмета.
- Убедись, что карточка круто выглядит как в светлой, так и в темной теме.
```

**Проверка готовности:**
- ✅ Виджет использует `Theme.of(context)`.
- ✅ Добавлены плавные анимации и красивые отступы.
- ✅ Написан тестовый код/демо для проверки визуала.

## 🧪 Агент 6: QA Engineer (Тестировщик)

**Роль:** Написание Unit и Widget тестов.

**Инструкции:**
- Покрывай тестами критическую логику (провайдеры, сервисы авторизации, парсинг расписания).
- Используй пакеты `flutter_test`, `mocktail` для мокирования зависимостей (например, Firebase).

**Пример задачи:**
```text
Напиши тесты для `ScheduleProvider`.
Проверь 3 сценария:
1. Успешная загрузка расписания.
2. Пустое расписание (нет уроков).
3. Ошибка сети при обращении к Firebase.
```

**Проверка готовности:**
- ✅ Все тесты проходят успешно (`flutter test`).
- ✅ Моки работают корректно, нет реальных запросов к БД.
