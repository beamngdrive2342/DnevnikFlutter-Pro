# Recovery Guide — DnevnikFlutter

Если потеряешь историю чата или начинаешь новую сессию с агентом — используй этот процесс.

## 📋 Порядок восстановления контекста

1. Прочитай `AGENTS.md` — правила работы агента
2. Прочитай `docs/PROJECT_CONTEXT.md` — что за проект, tech stack, фичи
3. Прочитай `docs/DECISIONS.md` — почему всё сделано именно так
4. Прочитай `docs/CHANGELOG.md` — что и когда менялось
5. Прочитай 2–3 последних файла в `logs/sessions/` — что делали в последние дни

## 🔄 Что нужно восстановить

- Что это за проект и для кого?
- На каком этапе разработки находится?
- Какие архитектурные решения приняты и почему?
- Что последнее было сделано?
- Какие файлы были изменены в последней сессии?
- Какие есть открытые задачи?
- Какие есть риски или технический долг?

## ⚡ Быстрый старт для нового агента

Скопируй этот промпт целиком в новый чат:

```
# Recovery Session — DnevnikFlutter

Я начинаю новую сессию с проектом DnevnikFlutter.
Помоги восстановить контекст из локальных файлов перед тем как продолжить работу.

## Файлы для чтения (в этом порядке)
1. AGENTS.md — правила для агентов, ограничения, инструкции
2. docs/PROJECT_CONTEXT.md — контекст проекта, tech stack, фичи
3. docs/DECISIONS.md — архитектурные решения с обоснованием
4. docs/CHANGELOG.md — история всех изменений
5. logs/sessions/ — логи последних 2–3 сессий (прочитай самые свежие файлы)

## Что нужно восстановить
1. **Карта проекта:** что это? зачем? для кого? на каком этапе?
2. **Tech stack:** какой стек, какие зависимости, какая архитектура?
3. **Последнее состояние:** что сделано последним? какие файлы менялись?
4. **Открытые задачи:** что планировалось сделать дальше?
5. **Ключевые решения:** почему Firestore REST, почему Vercel-прокси и т.д.?
6. **Риски:** что может сломаться, что нужно помнить?

## Инструкции
- Ничего не меняй в коде, только читай и отвечай
- Будь конкретным — избегай общих фраз
- Если какой-то файл непонятен или не найден — скажи
- После восстановления контекста — жди новую задачу

Готов? Начни читать файлы и дай краткий отчёт о состоянии проекта.
```

## 🗂️ Ключевые файлы проекта

| Файл | Назначение |
|------|-----------|
| `AGENTS.md` | Правила, ограничения, формат промптов |

| `docs/DECISIONS.md` | Архитектурные решения с обоснованием |
