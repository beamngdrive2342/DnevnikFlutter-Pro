# Рефакторинг dnevnik_app — набор задач для Claude (Opus)

Это набор самодостаточных тикетов. Каждый закрывает одну фазу **целиком** и рассчитан
на отдельную сессию Opus, стартующую «с холода» (без памяти о предыдущих сессиях).

## Как пользоваться (для вайб-кодера)

1. Открой новую сессию Claude Code в папке `src/` (это корень Flutter-проекта).
2. Скажи: **«Выполни задачу из `docs/refactor/phase-1-cleanup-and-tests.md` полностью, по шагам, с верификацией»**.
3. Дождись, пока агент закончит и покажет результат верификации (analyze + test + ручной прогон).
4. Сам проверь, что приложение запускается и работает. Только потом переходи к следующей фазе.

**Порядок строгий: 1 → 2 → 3 → 4.** Не прыгай вперёд.
Фаза 1 ставит тесты-страховку, без которых фазы 2–4 опасны.

## Фазы

| Файл | Что делает | Риск | Зависит от |
|---|---|---|---|
| `phase-1-cleanup-and-tests.md` | Удаляет мусор/мёртвый код, централизует даты, ставит настоящие unit-тесты | низкий | — |
| `phase-2-state-management.md` | Убирает глобальную статику, переводит состояние на Riverpod | средний | Фаза 1 |
| `phase-3-split-god-files.md` | Режет экраны по 800–1000 строк на модули | средний | Фаза 2 |
| `phase-4-firebase-sdk.md` | Переход на официальный Firebase SDK + Storage + Cloud Functions | высокий | Фаза 3 |

## Общие правила для агента (действуют во всех фазах)

- Рабочая директория: `src/`. Все команды (`flutter ...`) запускать оттуда.
- Комментарии/строки UI — на русском, как в существующем коде. Стиль и отступы — как в соседнем коде.
- Не добавлять `Co-Authored-By` в коммиты.
- Каждую фазу дробить на несколько маленьких коммитов (в тикете указано, как).
- «Готово» = `flutter analyze` без новых ошибок И `flutter test` зелёный И ручной прогон
  ключевого сценария из раздела «Верификация» этого тикета.
- Если по ходу вскрывается проблема вне scope фазы — не чинить молча, а записать в
  `docs/refactor/NOTES.md` и продолжить свою фазу.
