# ПРОМПТЫ ДЛЯ ЕЖЕДНЕВНОЙ РАБОТЫ (DnevnikFlutter)

Эти промпты предназначены для управления рабочими сессиями и задачами при разработке проекта DnevnikFlutter. Скопируйте нужный промпт и отправьте его агенту.

## 1. Начало сессии (/start-session)

Используйте этот промпт для старта новой рабочей сессии, чтобы погрузить агента в актуальный контекст проекта.

```markdown
# /start-session

Начни новую рабочую сессию в проекте DnevnikFlutter.

## Обязательные файлы для чтения:
1. AGENTS.md
2. docs/PROJECT_CONTEXT.md (если существует)
3. docs/DECISIONS.md (если существует)
4. Последний файл лога сессии из logs/sessions/

## Ожидаемый ответ:
1. **Контекст проекта** (1-2 предложения: что за проект и его цель).
2. **Текущий статус** (на каком этапе находимся, что было сделано последним).
3. **Открытые вопросы** (есть ли риски или блокеры).
4. **Первый шаг** (предложи, с чего мы можем начать работу прямо сейчас).

Пожалуйста, ничего не меняй в файлах до получения первой конкретной задачи.
```

## 2. Рабочая задача (/work-session)

Используйте этот шаблон для постановки конкретных задач агенту.

```markdown
# Рабочая задача

ЗАДАЧА: [Конкретно что нужно сделать, например: "добавь экран просмотра ДЗ"]

КОНТЕКСТ: [Дополнительные данные или условия, например: "ДЗ хранятся в Firestore: /homework/{date}/{subject}"]

ИСТОЧНИКИ:
- AGENTS.md — основные правила проекта
- docs/PROJECT_CONTEXT.md — контекст и логика
- src/lib/ — исходный код проекта
- Используй Skill: [например, brainstorming, frontend-design, systematic-debugging]

ОГРАНИЧЕНИЯ:
- Не выдумывай структуру данных и бизнес-логику — спрашивай, если не уверен.
- Следуй архитектуре: Riverpod 3 + go_router. Используй freezed для моделей.
- Не изменяй Firestore rules и pubspec.yaml без разрешения.

ФОРМАТ ОТВЕТА: [код / документ / план реализации / просто текст]

ОТЧЁТ:
- Что было запрошено (кратко)
- Что было сделано (пунктами)
- Какие файлы изменены
- Команды для проверки:
  `cd src && flutter analyze && flutter test && dart run build_runner build --delete-conflicting-outputs`
- Следующий шаг
```

## 3. Завершение сессии (/end-session)

Используйте этот промпт для корректного завершения работы, сохранения истории и документации проекта.

```markdown
# /end-session

Закрой текущую рабочую сессию. Пожалуйста, выполни следующие шаги:

1. Создай или обнови лог в `logs/sessions/YYYY-MM-DD.md` со следующей структурой:
   - Что запрашивалось в ходе сессии
   - Что было прочитано и изучено
   - Что было сделано (реализовано)
   - Какие файлы были изменены
   - Принятые решения (особенно архитектурные)
   - Открытые вопросы и блокеры
   - Следующий шаг для будущей сессии

2. Обнови `docs/CHANGELOG.md` (если был изменен исходный код).

3. Обнови `docs/DECISIONS.md` (если принимались новые архитектурные решения).

После успешного обновления файлов предоставь краткий отчет:
- Что было сделано за сессию (2-3 предложения)
- Какие файлы были обновлены (список)
- Команды для финальной проверки (например, `cd src && flutter analyze`)
- Следующий шаг для продолжения работы
```
