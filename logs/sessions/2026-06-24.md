## Что было сделано (реализовано)
- **Миграция на GitHub Raw CDN:** Все ~3400 картинок (Геометрия и Алгебра) были вынесены в публичный GitHub-репозиторий `dnevnik-assets`. APK уменьшился с ~400 МБ до ~111 МБ.
- **Внедрение `cached_network_image`:** Библиотека добавлена для надежного офлайн-кеширования готовых решений.
- **Мульти-фото для номеров:** Написана логика `_NumberImages`, которая проверяет наличие дополнительных частей картинок решения (через `http.head`) и сохраняет их счетчик в локальную память (`SharedPreferences`) для поддержки офлайна.
- **Офлайн вход (Фоллбэк):** В `auth_service.dart` и `auth_provider.dart` добавлен функционал сохранения документа класса в локальный кеш при успешном логине. При отсутствии сети приложение больше не выбрасывает пользователя из профиля, а загружает локальное расписание.
- **Аудит безопасности (Агент 2):** Проведена проверка `ai_service.dart`, `firestore_service.dart` и `auth_service.dart`. Подтверждено, что API ключи Gemini спрятаны за Vercel-proxy, а токены хранятся безопасно (`flutter_secure_storage`).
- **QA План (Агент 6):** Разработан план для добавления тестов с использованием `mocktail` (оформлен в артефакте).

## Принятые решения (особенно архитектурные)
- **Использование GitHub Raw в качестве CDN:** Вместо хранения сотен мегабайт картинок внутри приложения мы загружаем их "по воздуху".
- **Кэширование "из коробки":** Заменено базовое решение `Image.network` на `cached_network_image` для персистентного хранения.
- **Офлайн-персистентность:** Расписание класса кешируется в `SharedPreferences` так же, как и домашка, что позволяет полностью отвязать "режим чтения" дневника от интернета.
