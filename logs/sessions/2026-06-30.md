## Что было сделано (реализовано)
- **Локализована причина ошибки:** автоматические сборки Vercel из Git производились с корневой директорией проекта `rootDirectory` равной `.`. Поскольку папка `api/` находилась внутри `src/vercel-proxy/`, сборщик Vercel не находил функций в корне, собирал пустой проект и возвращал 404.
- **Исправлена конфигурация проекта:** через REST API Vercel параметр `rootDirectory` проекта `vercel-proxy` был обновлен на `src/vercel-proxy`.
- **Проведен новый деплой:** вызван повторный production-деплой, который успешно склонировал репозиторий, скомпилировал Edge-функцию `gemini.js` и опубликовал её.
- **Верификация:** произведен тестовый запрос к `https://vercel-proxy-delta-red.vercel.app/api/api/gemini`, подтвердивший успешный ответ `200 OK` от модели `gemini-3.1-flash-lite`.
