## Принятые решения (особенно архитектурные)

- **Удаление admin = full delete класса**: при удалении аккаунта администратора удаляется весь класс, все ДЗ, все участники (Вариант A, выбранный пользователем)
- **Privacy Policy на GitHub Pages**: документы в `docs/legal/`, включение Pages через Settings репозитория
- **Кнопка настроек вместо двух кнопок**: убрали дублирование (тема + выход) из TopBar, всё консолидировано в SettingsScreen

## Открытые вопросы и блокеры

- ⚠️ **Нужно настроить GitHub Pages**: в Settings репозитория включить Pages, указать ветку `main`, директорию `/docs`. После этого обновить URL Privacy Policy в `settings_screen.dart` (константа `_kPrivacyPolicyUrl`).
- ⚠️ **Нужно создать Release Keystore**: для публикации в Google Play нужен `upload-keystore.jks` и настройка `key.properties`. Инструкция: `keytool -genkey -v -keystore upload-keystore.jks -alias upload -keyalg RSA -keysize 2048 -validity 10000`.
- ⚠️ **Заполнить Data Safety Form** в Google Play Console: декларировать email (admin), Firebase Auth UID (анонимный), ДЗ, AI-запросы.
- ⚠️ **Заполнить Content Rating** (IARC questionnaire) в Google Play Console.

## Lessons Learned (Выводы для будущих агентов)

1. **Файлы с CRLF-окончанием не поддаются замене через инструменты edit** — используй Python-скрипт для замены с явным указанием `\r\n` в маркере. Это касается всех Dart-файлов в проекте.
2. **Всегда проверяй фактический API виджета** перед его использованием. `TopNotification` — это не класс, а просто функция `showTopNotification()`. Не выдумывай `ClassName.method()` API без чтения кода.
