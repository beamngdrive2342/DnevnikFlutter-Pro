## Принятые решения
- Логику «просрочено» и парсинг дедлайна держим в `app_date_utils.dart` как единый источник правды (см. DECISIONS 2026-07-13).
- Для тестируемости приватных чистых функций — паттерн `@visibleForTesting` (публичная обёртка/переименование), без изменения поведения.
- Инъекция `{DateTime? now}` в `isHomeworkExpired` — чтобы тесты были детерминированными, а не привязанными к системным часам.

## Lessons Learned (Выводы для будущих агентов)
1. **`flutter_dotenv 6.0.1` НЕ имеет `dotenv.testLoad()`** — метод из спеки Фазы 1 устарел. Актуальный API для тестов: `dotenv.loadFromString(envString: 'KEY=value')`. Записано в `docs/wiki/AGENTS_KNOWLEDGE.md`.
2. **`ClassSchedule` — глобальная мутабельная статика**, состояние течёт между тестами. В любом тесте, который её трогает, обязателен `setUp/tearDown` c `ClassSchedule.reset()`. Это же — главная причина, почему Фаза 2 (перевод на Riverpod) нужна.
3. **widget_test при пустых mock-prefs безопасен**: путь `AuthNotifier._checkSession` при `classId == null` не ходит в сеть и не трогает `flutter_secure_storage`, поэтому тест детерминирован. Нельзя использовать `pumpAndSettle` — на splash бесконечный `CircularProgressIndicator`.
