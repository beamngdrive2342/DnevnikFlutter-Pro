## Что было сделано (фиксы кэширования и скролла)
- **Оффлайн-кэш:** Полностью переписан `NetworkPhoto` (`network_photo.dart`). Теперь он использует `CachedNetworkImage` для удаленных URL, что позволяет сохранять фото ДЗ и оффлайн автоматически.
- **Оптимизация загрузки ГДЗ:** В `lesson_card.dart` исправлена функция `_checkImages()`. Теперь количество картинок для конкретного номера сохраняется локально (`SharedPreferences`), и при повторном открытии приложение не тратит 5 секунд на сетевые запросы `http.head`.
- **Скролл ГДЗ:** В `_showTextbookAnswers` виджет `ListView.builder` заменен на `SingleChildScrollView` + `Column`. Это решило проблему прыгающего скролла при загрузке картинок ГДЗ с неизвестной высотой.

## Принятые решения (особенно архитектурные)
- **Отложенная интеграция FCM:** Решено пока не внедрять Firebase Cloud Messaging из-за накладных расходов на серверную часть (Vercel Node.js + Firebase Admin). Все изменения за сегодня будут отменены пользователем.

## Открытые вопросы и блокеры
- Если в будущем захотим вернуть пуши, возможно, стоит использовать Vercel Edge Runtime или встроенные Firebase Functions, чтобы избежать проблем с компиляцией CommonJS/ESM.
- **Генерация чертежей для ГДЗ:** Сырой SVG оказался слишком хрупким и сложным для автоматической генерации точных геометрических фигур с перспективой. Как альтернатива рассматриваются: готовые картинки из GeoGebra/Figma, или интеграция математических движков (JSXGraph/GeoGebra WebView).

## Следующий шаг для будущей сессии
- Реализовать альтернативный подход к созданию чертежей (скорее всего, остановимся на загрузке готовых PNG/SVG картинок, нарисованных вручную в GeoGebra или других редакторах).
- Заняться интеграцией с электронным дневником, как планировалось изначально.

## Эксперимент с Markdown + SVG (Отменен)
- **Что делали:** Попытались заменить картинки-ответы из решебников на текст Markdown + LaTeX + SVG чертежи (чтобы обойти авторские права).
- **Результат:** Парсинг формул (LaTeX) через `flutter_math_fork` сработал отлично, но рисование сложной 3D геометрии (например, вписывание эллипсов в плоскости с перспективой) через сырой SVG код оказалось невероятно трудоемким и неточным.
- **Итог:** Решили отказаться от генерации сырого SVG. Код (`markdown_answer_viewer.dart` и зависимости `flutter_svg`, `flutter_markdown`) удалены пользователем. Решения по геометрии требуют другого подхода к чертежам.
